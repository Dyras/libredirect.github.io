A web extension that redirects YouTube, Twitter, Instagram... requests to alternative privacy friendly frontends and backends

  • YouTube => Invidious, Piped, Piped-Material, FreeTube, Yattee
  • Twitter => Nitter
  • Instagram => Bibliogram
  • TikTok => ProxiTok
  • Reddit => Libreddit, Teddit
  • Imgur => Rimgo
  • Wikipedia => Wikiless
  • YouTube Music => Beatbump
  • Spotify => Soju
  • Medium => Scribe
  • Peertube => SimpleerTube
  • LBRY/Odysee => Librarian
  • Search => SearXNG, SearX, Whoogle
  • Translate => SimplyTranslate, LingvaTranslate
  • Maps => OpenStreetMap
  • Send Files => Send

Why?

Normally you watch YouTube videos through youtube.com. Problem is everything you watch is tied to your identity (Google Account). Even if you didn't specify your real name, you're IP address is visible. Even if you use a VPN or the Tor Browser, you still have JavaScript enabled, a programming language that can get so much information about your browser and your system. It's like running a python program every time you visit a website. Just visit amiunique.org and see how much information is available to know about you, you're Platform, Timezone, Screen width, Screen height, Media devices, etc... All those may seem uncommon at first, but merging them together popups you out from the herd.
So why not disabling JavaScript? Great idea! Now YouTube is broken as it requires JavaScript to function ¯\_(ツ)_/¯

There's a way though to make a script that scraps data off of youtube.com and presents it in a nice simple webpage that doesn't require JavaScript. People did this already! There's a project called Invidious that doesn't require JavaScript for you to watch videos, ex: https://invidious.manerakai.com/watch?v=dQw4w9WgXcQ

Those are called alternative Frontends where you really are still using YouTube but with a different Interface, in our case a privacy friendly one.

The code is open source and you can run it on your home server or your personal computer. People did this already and there's a ton of Instances that are basically servers running Invidious in different parts of the world. Using an instance with multiple people helps to mix your data together as it exists from only one end. It's like using the same account on Netflix. Using multiple instances (a random one each time) will also mix your data more as it's now stored in multiple servers rather than in a single centralized one.

Using a random instance and changing links each time from ex:
https://youtube.com/watch?v=dQw4w9WgXcQ => https://invidious.manerakai.com/watch?v=dQw4w9WgXcQ
is a difficult process and best to be automated. Well, we already did that!
LibRedirect's main functionality is to make the use of those instances much easier for users and even unifies settings and cookies between them so you'll get a consistent interface while using multiple instnaces at the same time.

Same thing with Twitter, Instagram, Google Translate, Reddit, Imgur, TikTok and much more!

LibRedirect is a hub that connects normal users to privacy friendly frontends. It makes life much easier and privacy more convenient.

How to use it?

Download and install it.
Then type or click on any youtube link you want, ex: https://youtube.com/watch?v=dQw4w9WgXcQ
It will automatically redirect you to a random Invidious instance. You can fine tune your redirections in the settings page.
